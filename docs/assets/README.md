# Screenshots and demo assets

This directory holds the binary assets the README and docs reference.
The images are not generated by CI — they're captured manually after a
build that represents the current product surface, then committed.

## Files and what each should show

| File | Size target | Content |
|---|---|---|
| `pulse-dashboard.png` | 1600×1000 max | The Pulse page with runtime / provider / context / MCP / capabilities pills populated, at least one session in the recent threads list, and the Agent usage card with a non-zero value. |
| `chat-tool-delegation.png` | 1600×1000 max | An active thread mid-stream showing a sub-agent branch via `@research` or `@explore`: task-run card, transcript, tool calls, elapsed clock. |
| `capabilities-page.png` | 1600×1000 max | The Capabilities page with the built-in + custom sections visible, at least one MCP expanded so the tool list is on screen. |

## Capture guidelines

- Use the packaged macOS build against the isolated `smoke`-branded
  HOME (see `apps/desktop/tests/smoke-helpers.ts`) so screenshots
  don't leak a real project path, API key, or session title.
- Theme: system (the app respects OS theme). If your OS is in dark
  mode, take a matching dark-mode pair.
- Trim menu bar / dock / other chrome with a screenshot tool rather
  than cropping the PNG post-hoc — preserves the native hi-DPI
  density.
- Compress PNGs with `oxipng -o 4` or `pngcrush -brute` before
  committing; the CI docs build does not pipeline this.
- GIFs should not loop aggressively — one pass + final frame hold.
  If the captured GIF is over 5MB, re-encode at a lower frame rate
  (12 fps is usually fine for UI demos).
